# TogetherLog — Future Feature Specifications (Version 2+ Options)

This document archives planned optional and advanced features.  
They are not part of the MVP and will be implemented in later versions.

## 1. Map View (Open Source Based)
- Interactive world/region map using OpenStreetMap + Leaflet.
- Each entry with coordinates appears as a map marker.
- Click marker → open corresponding entry/page.
- Cluster markers for dense areas.
- Optional filters (tags, dates).

## 2. Story Slideshow
- Auto-advancing slideshow of entries.
- Fullscreen mode.
- Timing control (e.g., 5s per page).
- Optional transitions.
- Selectable subsets (e.g., a specific year or tag group).

## 3. Emotion-Color-Engine 2.0 (AI-Assisted)
- Enhanced color theme selection using:
  - Image mood detection
  - NLP on highlight text
- Allows more adaptive and expressive page themes.

## 4. AI-Assisted Smart Pages Improvements
- Smart tagging from photos.
- Suggested highlight text / titles.
- Recommended layouts based on content.
- Optional: User-provided API key for external LLM or vision models.

## 5. Sprinkles & Decorative Graphics
- Automatically inserted icons/symbols based on category.
- Examples:
  - Hearts for romance
  - Mountains for hiking
  - Beach icons for seaside entries
- Custom toggle to enable/disable per log or entry.

## 6. Polaroid & Collage Layout Controls
- User-selectable layout variations.
- Manual override of layout type chosen by Smart Pages.

## 7. Memory Widgets (Mobile Integration)
- Android/iOS widget that periodically displays random past memories.
- Deep link to the entry in the web app.

## 8. Relationship / Group Progress System (Gamification)
- Optional toggle per log.
- XP system based on tags and activities.
- Category-based "levels":
  - Travel Level
  - Adventure Level
  - Romance Level
- Visual indicators on user profile.

## 9. Heatmap Visualization
- Map heatmap of visited places.
- Color intensity based on number of memories.
- Year-based filters.

---

## Notes
These features are **not required** for the initial release, but the V1 data model and architecture should allow easy integration without breaking changes.
